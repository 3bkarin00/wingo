# R0 Findings — P2 (Sections + OML loft)

## Real OCP loft API (confirmed by probe, not assumed)

- Build a **closed** section wire from canonical airfoil points as:
  `spline = cq.Edge.makeSpline([Vector...])` (the body) + `cq.Edge.makeLine(last, first)`
  (a straight edge across the blunt TE), assembled via `cq.Wire.assembleEdges`.
  Result is a closed `Wire`.
- Loft: **`cq.Solid.makeLoft(list_of_wires, ruled=False)` → `Solid`**. `ruled=False`
  gives a smooth (splined) skin between sections; `ruled=True` would give
  straight (developable) transitions.
- Watertightness check: `BRepCheck_Analyzer(solid.wrapped).IsValid()` **AND**
  every `solid.Shells()` reports `.wrapped.Closed()`. Both required.

## ⚠️ The trap this probe caught (drives a gate design decision)

Rolling ONE section's start point (same shape, different starting index) before
lofting produced a **badly distorted, twisted solid — volume 0.071 vs 0.341 for
the aligned loft, ~5× wrong — yet `IsValid()` still returned `True`.**

Implications:
1. **`IsValid()` is necessary but NOT sufficient** for a correct OML. The P2
   gate MUST also assert the volume-conservation band (±3% of the analytic
   planform×mean-thickness estimate, per §9 P2) — a twisted loft passes
   watertightness but fails volume. This is exactly why the plan pairs the two.
2. **Sections must share a consistent point order and start vertex.** Our
   canonical airfoil representation (TE→upper→LE→lower→TE, identical odd
   resample count for every section) satisfies this BY CONSTRUCTION: every
   section starts at the TE upper point with the same index structure, so
   loft correspondence is aligned without extra work. The loft code must not
   re-order or re-index sections per-station.

## ⚠️ Refinement from the P2 volume diagnostics (spline → polygon+ruled)

The initial probe used SPLINE wires + `ruled=False`. Volume diagnostics on the
golden configs then showed two problems that changed the construction choice:

1. A spline loft (`ruled=False`) **bulges ~3.1% outward** between sections vs
   the faithful ruled/prismatoid volume — landing right on the ±3% gate limit,
   and its volume is **unstable with section count** (9.97M→10.80M for minimal
   as sections 2→40). Not a safe ground truth.
2. `ruled=True` with SPLINE wires is worse (~20% off) — it lofts only the
   wires' two EDGE endpoints, not the airfoil points.

**Decision: build each section wire as a closed POLYGON (`cq.Wire.makePolygon(
verts, close=True)`) and loft with `ruled=True`.** This connects all N
corresponding section vertices, so the loft volume equals the analytic
prismatoid to <0.3% (minimal 0.17%, full_example 0.28%), is stable, and yields
planar facets that are robust for the boolean cuts in P4+. The blunt-TE edge is
the polygon's closing segment (last point → first point). Watertight (valid +
closed shell) confirmed on both golden configs.

## Implications for P2 construction (§8.2–8.3)

- Scale each unit-chord canonical section by chord_mm, apply twist (rotation
  about the declared `twist_axis_xc` chord fraction), then place in 3D with
  per-segment dihedral/sweep. Point ordering is preserved throughout, so the
  loft stays aligned.
- Mirror for full span by reflecting the half-span sections across the root
  plane (Y=0) — reflection reverses winding, so the mirrored loft's wire
  orientation must be handled consistently (verify watertight after mirror).

- validity: BRepCheck IsValid = True; shell count = 1; all shells Closed = True (watertightness = valid + closed shell)

- CONCLUSION: build closed wires (spline + TE edge), keep consistent point order/start across sections, loft with `cq.Solid.makeLoft(ruled=False)`, verify watertight via BRepCheck_Analyzer.IsValid() + Shell.Closed().
