# R0 Findings — P3 (Reference geometry)

The uncommitted P3 work built `backend/geometry/reference.py` (spar ruled
surfaces, rib planes, hinge axes, hardpoints) directly against `cq.Face.
makeRuledSurface` / `cq.Shell.makeShell` / `BRepClass3d_SolidClassifier`
without a documented R0 probe first (§0.1 step 2 requires one for every new
third-party boundary a phase touches). Backfilling it here — and it caught a
real bug in the gate as originally written (see below and changelog.md).

## probe_ocp_reference.py

- test solid: 100x100x100 box centered at origin, volume=999999.9999999999
- `cq.Face.makeRuledSurface(edge, edge)` -> `Face`; `cq.Shell.makeShell([face])` -> `Shell`, isValid=True
- BRepClass3d_SolidClassifier at center (0.0, 0.0, 0.0): state=TopAbs_State.TopAbs_IN (IN=TopAbs_State.TopAbs_IN, ON=TopAbs_State.TopAbs_ON)
- BRepClass3d_SolidClassifier at near-surface (49.999, 0.0, 0.0): state=TopAbs_State.TopAbs_IN (IN=TopAbs_State.TopAbs_IN, ON=TopAbs_State.TopAbs_ON)
- WRONG APPROACH: `BRepExtrema_DistShapeShape(vertex_at_center, SOLID).Value()` = 0.0000 mm — 0 for ANY interior point (vertex-in-volume is trivially 'touching'), useless as a margin metric.
- box.Shells() -> 1 shell(s)
- `BRepExtrema_DistShapeShape(vertex_at_center, box.Shells()[0]).Value()` = 50.0000 mm (expected: center=50.0 [half box width], near-surface=0.1)
- `BRepExtrema_DistShapeShape(vertex_at_near-surface, box.Shells()[0]).Value()` = 0.1000 mm (expected: center=50.0 [half box width], near-surface=0.1)
- CONCLUSION: distance-to-SOLID is always 0 for interior points — useless. Must compute `BRepExtrema_DistShapeShape(vertex, solid.Shells()[0].wrapped).Value()` (distance to the boundary SHELL) to get the true point-to-surface margin. Combine with the point-in-solid classifier (must be IN, not just close to the shell from outside) for a complete containment-with-margin check.

## Why this matters for the P3 gate (§9 pass criteria)

The plan's actual requirement is "containment sampled at ≥50 stations along
each axis **with margin ≥ sandwich stack**" (F5: a twisted hinge axis can
approach the OML skin mid-span even while nominally "inside"). The gate as
first drafted only checked `BRepClass3d_SolidClassifier` state IN/ON — which
says nothing about *how far* inside a point is. A hinge axis sitting exactly
at the skin (zero clearance) would have passed that check. The fix: pair the
classifier (must be IN, not approaching from outside) with
`BRepExtrema_DistShapeShape(vertex, oml.Shells()[0].wrapped).Value() >=
stack_mm`, using the real distance-to-shell API confirmed above.
